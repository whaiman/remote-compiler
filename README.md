# RGCC - Remote GCC Compiler System

RGCC is a robust, distributed compilation utility designed for remote building of C/C++ projects. Engineered with a zero-bloat architecture, it securely offloads the compilation compute from resource-constrained developer environments (e.g., lightweight laptops, Raspberry Pi, or Android/Termux) to a dedicated, high-performance remote build server.

## Key Features

- **End-to-End Encryption**: AES-256-GCM protects your source code and binaries during transmission.
- **Secure Key Exchange (ECDH + HKDF)**: Dynamic Elliptic-Curve Diffie-Hellman handshake for generating ephemeral session keys without transmitting them over the network.
- **Stateless Session Tickets**: Server memory is protected from DDoS and memory leaks. The server does not maintain session states in its RAM.
- **MITM Protection**: Automatic mixing of a Pre-Shared `auth_token` into the key derivation process to block Man-In-The-Middle attacks.
- **Lightweight & Highly Portable:** Designed with a minimal footprint to run smoothly on low-power devices like Raspberry Pi, old laptops, or even Android via Termux.

> [!NOTE]
> We intentionally avoided heavy frameworks (like FastAPI or Pydantic) that require complex C/Rust toolchains during installation. If a device can run standard Python, it can run RGCC.

> [!TIP]
> While the server itself is lightweight, compilation speed depends entirely on your project's complexity and the hardware it's running on. Large projects will still take time on mobile CPUs.

## Architecture

The project is split into two independent installable packages:

| Package       | Command | Purpose                                                                          |
| ------------- | ------- | -------------------------------------------------------------------------------- |
| `rgcc-client` | `rgcc`  | Developer CLI - collects sources, encrypts, sends to server, retrieves artifacts |
| `rgcc-server` | `rgccd` | Build daemon - receives payload, compiles, returns artifacts                     |

Both share a common `rgcc/core/` layer (cryptography, config, manifest, checksum).

## Installation

Install only what you need on each machine.

### Developer Node (Client)

```bash
pip install "rgcc-client @ git+https://github.com/whaiman/remote-compiler.git#subdirectory=packages/rgcc-client"
```

### Build Node (Server)

```bash
pip install "rgcc-server @ git+https://github.com/whaiman/remote-compiler.git#subdirectory=packages/rgcc-server"
```

### Local Development (both packages)

```bash
git clone https://github.com/whaiman/remote-compiler.git
cd remote-compiler
make install-dev
```

## 1. Build Node Setup (Server)

Ensure `gcc`, `g++`, or `clang` are available in the system path, then start the daemon:

```bash
rgccd
```

On first boot, the server generates `.rgcc/server.config.yaml` with a secure auth token:

```yaml
# .rgcc/server.config.yaml
server:
  auth_token: <auto-generated-url-safe-base64-token> # copy this to the client
  host: 0.0.0.0
  port: 4444

compilers:
  clang++:
    default_args: ["-fcolor-diagnostics"]
    platforms:
      win64:
        target: "x86_64-w64-mingw32"
        sysroot: "/usr/x86_64-w64-mingw32"
        args: ["-static", "-fuse-ld=lld", "-static-libgcc", "-static-libstdc++"]
  clang:
    platforms:
      win64:
        target: "x86_64-w64-mingw32"
        args: ["-static", "-fuse-ld=lld"]
```

Override host/port at runtime if needed:

```bash
rgccd --host 0.0.0.0 --port 5000
rgccd --reload  # development mode with auto-restart
```

## 2. Developer Node Setup (Client)

Create `.rgcc/client.config.yaml` and point it to the build server:

```yaml
# .rgcc/client.config.yaml
client:
  endpoint: http://192.168.1.50:4444
  auth_token: <paste-token-from-server.config.yaml>
```

### CLI Operations

#### Version

```bash
rgcc --version
rgcc -v

rgccd --version
rgccd -v
```

#### Rapid Iteration

The CLI recursively tracks local headers and orchestrates the payload.

```bash
# Standard compilation using ambient platform defaults
rgcc compile sample/main.cpp

# Target specific output binaries and language standards
rgcc compile sample/main.cpp -o my_renderer --std c++20

# Override target heuristics for cross-compilation (linux -> win64)
rgcc compile sample/main.cpp --platform win64
```

#### Interactive Orchestration (`-i`)

Leverage the TUI to mutate toolchains dynamically and hydrate persistent configuration logic (`build.json`).

```bash
rgcc compile sample/main.cpp -i
```

#### Persistent State Management (`build.json`)

For complex topologies, declare state within a JSON-serialized manifest. Generate via `rgcc init` or interactive mode.

```json
{
  "compiler": "clang++",
  "standard": "c++20",
  "entry_point": "sample/main.cpp",
  "output": "main.exe",
  "platform": "win64",
  "flags": ["-Wall", "-O3", "-static"],
  "defines": ["DEBUG=1"]
}
```

_Note: Transient lifecycle attributes (SHA-256 checksum logic, UTC chron timestamps, recursive source graphs) are extrapolated natively at runtime to enforce deterministic payloads._

#### Pipeline Utilities

```bash
# Abstract syntax evaluation (no linkage)
rgcc compile sample/main.cpp --compile-only

# Local packaging validation (no wire transfer)
rgcc compile sample/main.cpp --dry-run
```

All resultant compilation artifacts (`.out` / `.exe`, STDOUT logs, telemetry manifests) are automatically pipelined back to the developer node and extracted securely into the local `dist/` directory (leveraging TarSlip mitigation).

## Docker

RGCC ships a Docker image for the **build server**. It is built from the repository root using `Dockerfile.server`.

### Images Overview

| Image         | Dockerfile          | Entrypoint | Purpose                                         |
| ------------- | ------------------- | ---------- | ----------------------------------------------- |
| `rgcc-server` | `Dockerfile.server` | `rgccd`    | Build daemon with GCC/Clang/MinGW pre-installed |

Multi-stage build: dependencies compiled in `python:3.11-slim`, then copied into a clean runtime image with `gcc`, `g++`, `clang`, and `mingw-w64`.

### Run with Docker

```bash
# Build locally
make docker-build

# Prepare config
cp .rgcc/server.config.yaml.example .rgcc/server.config.yaml
# set a strong auth_token in .rgcc/server.config.yaml

# Run
make docker-run-server

# Or manually
docker run -d \
  --name rgcc-server \
  -p 4444:4444 \
  -v "$(pwd)/.rgcc:/app/.rgcc:ro" \
  rgcc-server:latest

docker logs -f rgcc-server
```

### Pull Pre-built Images from GHCR

Pre-built images are published to GitHub Container Registry automatically on every push to `main` and on every version tag:

```bash
# Pull the latest server build
docker pull ghcr.io/whaiman/remote-compiler-server:latest

# Pin to a specific release
docker pull ghcr.io/whaiman/remote-compiler-server:v2.0.0
```

## Security Considerations

- **Experimental Software**: This tool has not undergone formal security auditing. Use at your own risk in production environments.
- **Network Exposure**: The server binds to `0.0.0.0` by default. Use firewall rules to restrict access.
- **Token Storage**: `auth_token` is stored in plain text in `.rgcc/`. Ensure this directory has `chmod 700` permissions.
- **Compiler Whitelist**: Only pre-approved compilers (`gcc`, `clang`, etc.) are executable on the server to mitigate RCE risks.
